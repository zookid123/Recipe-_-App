# 개발 이력 & 향후 계획

## 개발 이력

### 역할 분담

| 이름 | 역할 |
|------|------|
| 강민권 | 안드로이드 구현, DB구축 |
| 김윤상 | UI디자인, DB구축 |
| 박성엽 | UI디자인, 안드로이드 구현 |
| 김진석 | UI디자인, 안드로이드 구현, 회의록 작성 |

### 개발 단계

| 단계 | 주요 작업 |
|------|----------|
| 초기 세팅 | Firebase 연동, 기본 화면 구성 (홈, 레시피, 마이페이지) |
| 레시피 기능 | 레시피 필터·정렬, 검색 기능, DB구축 |
| 개인화 기능 | 북마크, 최근 본 레시피, 활동 내역 |
| 커뮤니티·채팅 | 1:1 쪽지(채팅), 커뮤니티 게시글·댓글 |
| 관리자·업적 | 관리자 패널, 운영자 칭호·업적 |
| 알림 연동 | 댓글 알림·유통기한 임박 알림 실제 동작 연동, 알림 설정 ON/OFF |

## 향후 개발 로드맵

### P0 — 개인화 기능 강화
- 즐겨찾기(북마크) 저장 UI 개선
- 최근 본 레시피 활동 내역 고도화

### P1 — 커뮤니티 활성화
- 요리 자랑 게시판 전용 카테고리
- 레시피별 요리 후기 사진 리뷰 기능
- 냉장고 파먹기 검색 정확도 향상 (Firestore 인덱싱 적용)

### P2 — 사용자 편의 기능
- 재료 장바구니 리스트 (조리 시 필요한 재료 체크 목록)
- 조리 전용 모드 고도화 (타이머 연동, 음성 안내)
- 검색 고도화 (0kcal 필터링, Firestore 인덱스 기반 서버 검색)

## 기술적 개선 과제

| 항목 | 내용 |
|------|------|
| 환경 변수 관리 | API 키·Firebase 키 하드코딩 → `.env` 또는 CI/CD 시크릿으로 이동 |
| 검색 고도화 | 클라이언트 사이드 필터링 → Firestore 인덱스 기반 서버 검색 |
| 상태 관리 | StatefulWidget → Riverpod 또는 Bloc 도입 검토 |
| 테스트 | 단위 테스트·위젯 테스트 커버리지 확보 |
